Documentation

AgentStatic generates pure HTML + CSS static sites. This guide covers everything you need to get started.

Quick Start

  1. Edit brand.json with your brand name and primary color
  2. Replace assets/logo.svg with your logo
  3. Update the pages in pages/ with your content
  4. Run npm run build to generate your site
  5. Deploy the dist/ folder anywhere

Project Structure

my-site/
├── brand.json        # Your brand config (edit this first!)
├── pages/            # Your HTML pages
│   ├── index.html    # Homepage
│   ├── about.html    # About page
│   └── docs.html     # This page
├── partials/         # Reusable HTML fragments
├── elements/         # Custom element styles
├── utilities/        # Utility CSS classes
├── assets/           # Static files (logo, images)
└── dist/             # Built output (deploy this)

Customizing Your Brand

Everything starts with brand.json:

{
  "name": "Your Company",
  "tagline": "Your tagline here",
  "colors": {
    "primary": "#22c55e"
  }
}

The build system generates a complete color palette from your primary color—shades from 50 to 950, plus semantic colors for backgrounds, text, and accents.

Available Elements

AgentStatic includes pre-styled custom HTML elements:

  • <site-hero> — Full-width hero section (variants: centered, split; themes: dark, light)
  • <site-features> — Feature grid (variants: 2-col, 3-col, 4-col)
  • <site-pricing> — Pricing tiers (use featured attribute to highlight)
  • <site-faq> — FAQ accordion using native <details>
  • <site-cta> — Call-to-action section (themes: dark, light)

Using Partials

Partials are reusable HTML fragments. Include them with <include-name>:

<include-head title="Page Title" description="Meta description" />

<include-nav logo="/assets/logo.svg" siteName="My Site">
  <a slot="links" href="/">Home</a>
  <a slot="links" href="/about.html">About</a>
  <a slot="actions" href="/signup" class="button">Sign Up</a>
</include-nav>

<include-footer siteName="My Site" tagline="Your tagline">
  <nav slot="links">
    <h4>Company</h4>
    <a href="/about.html">About</a>
  </nav>
</include-footer>

Creating New Elements

To add a custom element:

  1. Create a folder in elements/ (e.g., elements/site-testimonials/)
  2. Add a README.md documenting usage and variants
  3. Add a styles.css with your element styles
  4. Run npm run build — your styles are automatically included

Commands

# Build once
npm run build

# Build and watch for changes
npm run dev

# Serve the built site locally
npm run serve

Deployment

The build outputs plain HTML and CSS to dist/. Deploy anywhere:

  • Netlify — Connect repo, build command: npm run build
  • Vercel — Auto-detects static sites
  • GitHub Pages — Push dist/ to gh-pages branch
  • Any web server — Upload the dist/ folder

Troubleshooting

Partial Not Expanding

  • Check filename matches: <include-foo> needs partials/foo.html
  • Check for typos in attribute names

Styles Not Applying

  • Verify element name matches CSS selector
  • Check CSS is in elements/*/styles.css or utilities/
  • Run build to regenerate dist/styles.css

Colors Wrong

  • Check brand.json syntax (valid JSON)
  • Run build to regenerate tokens/colors.css

For detailed element documentation, check the elements/*/README.md files in the repository.
